# 管理后台登录登出接口文档

## 📋 接口概览

| 接口名称 | 请求方式 | 接口路径 | 端口 | 认证要求 |
|---------|---------|---------|------|---------|
| 管理员登录 | POST | `/api/admin/login` | 8081 | 无需认证 |
| 获取管理员信息 | GET | `/api/admin/profile` | 8081 | 需要Token |
| 退出登录 | POST | `/api/admin/logout` | 8081 | 无需认证 |

---

## 🔐 1. 管理员登录接口

### 接口信息
- **URL**: `POST /api/admin/login`
- **端口**: 8081
- **认证**: 无需认证
- **Content-Type**: `application/json`

### 请求参数
```json
{
  "username": "管理员用户名",
  "password": "登录密码"
}
```

**参数说明：**
| 参数名 | 必填 | 类型 | 说明 |
|--------|------|------|------|
| username | 是 | String | 管理员用户名（手机号） |
| password | 是 | String | 登录密码 |

### 成功响应
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzUxMiJ9.xxxxxx",
    "userInfo": {
      "id": 1,
      "username": "admin",
      "nickname": "超级管理员",
      "avatar": null,
      "role": "super_admin",
      "lastLoginAt": "2026-03-05T13:30:00",
      "status": 1,
      "createAt": "2026-03-04T17:00:00",
      "updateAt": "2026-03-05T13:30:00"
    }
  }
}
```

### 错误响应
```json
{
  "code": 500,
  "message": "用户名不能为空",
  "data": null
}
```

**常见错误信息：**
- "用户名不能为空"
- "密码不能为空" 
- "用户不存在"
- "密码错误"
- "账户已被禁用"

### 测试示例
```bash
curl -X POST http://localhost:8081/api/admin/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

## 👤 2. 获取管理员信息接口

### 接口信息
- **URL**: `GET /api/admin/profile`
- **端口**: 8081
- **认证**: 需要JWT Token
- **Authorization**: `Bearer {token}`

### 请求头
```
Authorization: Bearer eyJhbGciOiJIUzUxMiJ9.xxxxxx
```

### 成功响应
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1,
    "username": "admin",
    "nickname": "超级管理员",
    "avatar": null,
    "role": "super_admin",
    "lastLoginAt": "2026-03-05T13:30:00",
    "status": 1,
    "createAt": "2026-03-04T17:00:00",
    "updateAt": "2026-03-05T13:30:00"
  }
}
```

### 错误响应
```json
{
  "code": 500,
  "message": "未登录",
  "data": null
}
```

**常见错误信息：**
- "未登录"
- "token无效"
- "用户不存在"

### 测试示例
```bash
curl -H "Authorization: Bearer your_jwt_token" \
  http://localhost:8081/api/admin/profile
```

## 🚪 3. 退出登录接口

### 接口信息
- **URL**: `POST /api/admin/logout`
- **端口**: 8081
- **认证**: 无需认证（客户端自行清除token）

### 说明
由于JWT是无状态令牌，服务端不需要特殊处理退出逻辑。客户端只需清除本地存储的token即可。

### 成功响应
```json
{
  "code": 200,
  "message": "退出成功",
  "data": null
}
```

### 测试示例
```bash
curl -X POST http://localhost:8081/api/admin/logout
```

## 📱 客户端使用流程

### 1. 登录流程
```
1. 用户输入用户名和密码
2. 调用 POST /api/admin/login
3. 保存返回的JWT token
4. 后续请求在Header中携带 Authorization: Bearer {token}
```

### 2. 保持登录状态
```
1. 每次请求携带有效的JWT token
2. 可调用 GET /api/admin/profile 验证token有效性
3. token过期时重新登录
```

### 3. 退出登录
```
1. 客户端清除本地存储的token
2. 可选择性调用 POST /api/admin/logout
3. 重定向到登录页面
```

## ⚙️ 技术说明

### 认证机制
- 使用JWT (JSON Web Token) 进行身份认证
- Token有效期：24小时
- Token存储：客户端本地存储

### 安全措施
- 密码使用BCrypt加密存储
- JWT签名验证
- 请求频率限制
- 跨域资源共享(CORS)支持

### 错误处理
- 统一的错误响应格式
- 详细的错误信息提示
- HTTP状态码规范使用
