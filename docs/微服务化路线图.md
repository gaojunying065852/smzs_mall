# 微服务化演进路线图

## 当前架构（已完成）
```
smzs-mall-parent/
├── smzs-mall-common/          # 公共模块
├── smzs-mall-app/             # 小程序服务 (port: 8080)
└── smzs-mall-manage/          # 管理后台服务 (port: 8081)
```

## 第一阶段：服务拆分（建议）
将现有模块进一步拆分为独立的微服务：

### 用户服务 (user-service)
- 端口: 8082
- 职责: 用户管理、认证授权
- 接口: /api/users/**

### 商品服务 (product-service)  
- 端口: 8083
- 职责: 商品管理、分类管理
- 接口: /api/products/**, /api/categories/**

### 订单服务 (order-service)
- 端口: 8084
- 职责: 订单管理、支付处理
- 接口: /api/orders/**

### 文件服务 (file-service)
- 端口: 8085
- 职责: 文件上传、存储管理
- 接口: /api/upload/**

## 第二阶段：基础设施完善

### 服务注册与发现
- Eureka Server 或 Nacos
- 各服务注册到服务中心

### 配置中心
- Spring Cloud Config 或 Nacos Config
- 统一管理各服务配置

### API网关
- Spring Cloud Gateway
- 统一路由、鉴权、限流

### 链路追踪
- Zipkin + Sleuth
- 分布式调用链监控

## 部署架构演进

### 当前：单机Docker部署
```yaml
version: '3.8'
services:
  postgres: {...}
  redis: {...}
  app-service: {...}    # 8080
  manage-service: {...} # 8081
  nginx: {...}
```

### 目标：Kubernetes集群部署
```
Namespace: smzs-mall
├── ConfigMap: 配置管理
├── Service: 服务发现
├── Deployment: 服务部署
├── Ingress: 外部访问
└── PVC: 持久化存储
```

## 迁移步骤建议

1. **第一阶段**（1-2个月）
   - 拆分用户服务和商品服务
   - 引入服务间通信（Feign/Ribbon）
   - 添加基础监控

2. **第二阶段**（2-3个月）  
   - 拆分订单服务和文件服务
   - 引入消息队列（RabbitMQ/RocketMQ）
   - 完善分布式事务处理

3. **第三阶段**（3-4个月）
   - 上云部署（阿里云/AWS）
   - 引入容器编排（K8s）
   - 完善DevOps流程
