# 本地编译 + 阿里云Docker构建说明

## 概述

本文档介绍了如何在本地编译项目，然后将编译产物上传到阿里云服务器进行Docker镜像构建的完整流程。

## 整体架构变化

### 原有方案（使用Nexus）
```
本地代码 → Nexus私服 → 阿里云Docker构建（多阶段）
```

### 新方案（本地编译）
```
本地编译 → 上传编译产物 → 阿里云Docker构建（单阶段）
```

## Dockerfile变更说明

### 原Dockerfile（多阶段构建）
```dockerfile
# 多阶段构建 - 构建阶段
FROM maven:3.8.4-amazoncorretto-8 AS builder
WORKDIR /build
COPY . .
RUN mvn install -pl :smzs-mall-parent -am -DskipTests
RUN mvn clean package -DskipTests -pl :smzs-mall-app -am

# 运行阶段
FROM amazoncorretto:8-alpine
COPY --from=builder /build/smzs-mall-app/target/*.jar app.jar
```

### 新Dockerfile（单阶段构建）
```dockerfile
# 单阶段运行镜像 - 使用预编译的JAR文件
FROM amazoncorretto:8-alpine
WORKDIR /app
COPY target/*.jar app.jar  # 直接复制预编译的JAR文件
```

## 构建流程

### 1. 本地准备工作

#### 环境要求
- Java 8+
- Maven 3.6+
- Git

#### 执行本地编译
```bash
# Windows PowerShell
.\build-for-aliyun.ps1

# Linux/macOS Bash  
./build-for-aliyun.sh
```

执行成功后会生成：
- `smzs-mall-aliyun-build-YYYYMMDD-HHMMSS.tar.gz` 压缩包
- 内含完整的构建所需文件

### 2. 阿里云服务器准备

#### 环境要求
- Docker已安装并运行
- 已登录阿里云容器镜像服务

#### 登录阿里云镜像仓库
```bash
docker login registry.cn-hangzhou.aliyuncs.com
# 输入用户名和密码
```

### 3. 服务器端构建

#### 上传并解压构建包
```bash
# 上传文件到服务器
scp smzs-mall-aliyun-build-*.tar.gz user@your-server:/home/user/

# SSH登录服务器
ssh user@your-server

# 解压文件
tar -xzf smzs-mall-aliyun-build-*.tar.gz
cd aliyun-build
```

#### 配置构建参数
编辑 `build-images.sh` 文件：
```bash
# 修改这些变量为你的实际配置
REGISTRY="registry.cn-hangzhou.aliyuncs.com"  # 你的阿里云镜像仓库地址
NAMESPACE="your-namespace"                     # 你的命名空间
```

#### 执行构建
```bash
chmod +x build-images.sh
./build-images.sh
```

## 目录结构说明

构建包解压后的目录结构：
```
aliyun-build/
├── smzs-mall-app/           # 小程序服务
│   ├── Dockerfile          # 简化版Dockerfile
│   └── target/
│       └── *.jar           # 预编译的JAR文件
├── smzs-mall-manage/        # 管理后台服务
│   ├── Dockerfile          # 简化版Dockerfile
│   └── target/
│       └── *.jar           # 预编译的JAR文件
├── build-images.sh         # 阿里云构建脚本
├── .dockerignore           # Docker忽略文件
└── pom.xml                 # 项目配置文件
```

## 优势对比

| 方面 | 原方案(Nexus) | 新方案(本地编译) |
|------|---------------|------------------|
| 构建时间 | 较长（需下载依赖） | 较短（使用预编译文件） |
| 网络依赖 | 高（需访问Nexus） | 低（只需上传编译产物） |
| 复杂度 | 高（需维护Nexus） | 低（简化流程） |
| 成本 | 需要Nexus服务器资源 | 无需额外服务器 |
| 可靠性 | 依赖Nexus稳定性 | 不依赖外部服务 |

## 注意事项

### 安全考虑
1. 构建包包含编译后的JAR文件，请妥善保管
2. 阿里云镜像仓库地址和认证信息不要泄露
3. 建议使用私有镜像仓库

### 性能优化
1. 本地编译时可以使用多线程：`mvn clean package -T 4 -DskipTests`
2. 阿里云服务器建议使用较高配置以加快构建速度
3. 可以启用Docker构建缓存优化后续构建

### 故障排除
1. 如果构建失败，检查Docker是否正常运行
2. 确认阿里云镜像仓库权限配置正确
3. 验证网络连接是否稳定

## 自动化建议

可以将此流程集成到CI/CD系统中：
1. Jenkins Pipeline自动执行本地编译
2. 自动上传到指定服务器
3. 触发远程构建脚本执行

这样可以实现完全自动化的部署流程。
