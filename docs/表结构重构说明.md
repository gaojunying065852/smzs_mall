# 数据库表结构重构说明

## 重构内容

本次重构为所有数据表添加了统一的基础字段，遵循标准的企业级开发规范。

## 新增基础字段

| 字段名 | 类型 | 是否可空 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| creater | BIGINT | 可空 | 无 | 创建人ID |
| updater | BIGINT | 可空 | 无 | 修改人ID |
| create_at | TIMESTAMP | 非空 | CURRENT_TIMESTAMP | 创建时间 |
| update_at | TIMESTAMP | 非空 | CURRENT_TIMESTAMP | 更新时间 |
| delete | SMALLINT | 非空 | 0 | 逻辑删除标志：0正常，1删除 |

## 索引优化

### 索引优化
- 移除了逻辑删除相关的WHERE条件和索引
- 保留了核心业务字段索引
- 特殊条件索引仍保留WHERE子句（如推荐商品、热门商品等）

### 索引列表
```sql
-- 核心业务索引
CREATE INDEX idx_users_phone ON users(phone);
CREATE INDEX idx_users_open_id ON users(open_id);
CREATE INDEX idx_products_category ON products(category_id);
CREATE INDEX idx_addresses_user ON addresses(user_id);
-- ... 其他核心索引

-- 条件索引
CREATE INDEX idx_products_recommend ON products(is_recommend) WHERE is_recommend = TRUE;
CREATE INDEX idx_products_hot ON products(is_hot) WHERE is_hot = TRUE;
CREATE INDEX idx_sms_codes_phone_expire ON sms_codes(phone, expire_time) WHERE is_used = FALSE;
```

## Java实体类变更

### BaseEntity基类
创建了 `BaseEntity` 抽象类，包含所有基础字段和通用方法：
- 自动填充创建时间和更新时间
- 逻辑删除默认值设置
- JPA生命周期回调方法

### 实体类继承
所有实体类现在继承 `BaseEntity`，避免代码重复：
```java
@Entity
@Table(name = "users")
public class User extends BaseEntity {
    // 业务字段...
}
```

## 查询说明

### 基础查询
```sql
-- 基础查询
SELECT * FROM users;

-- 带条件查询
SELECT * FROM products WHERE category_id = 1;
```

### 逻辑删除（如需使用）
```sql
-- 软删除
UPDATE users SET delete = 1, updater = ? WHERE id = ?;

-- 查询未删除数据
SELECT * FROM users WHERE delete = 0;
```

## 迁移建议

1. **备份数据**：迁移前务必备份现有数据
2. **分步执行**：建议先在测试环境验证
3. **数据迁移**：如有历史数据，需要设置合理的默认值
4. **应用适配**：更新DAO层和Service层代码适配新字段

## 注意事项

- SQL脚本使用PostgreSQL标准语法
- 注释采用`COMMENT ON COLUMN`语法，兼容PostgreSQL
- 所有时间字段使用TIMESTAMP类型
- 逻辑删除字段使用SMALLINT类型，节省存储空间

## 性能影响

- ✅ 查询性能提升：通过 `WHERE delete = 0` 过滤提高查询效率
- ✅ 存储开销：每表增加5个字段，存储空间略有增加
- ✅ 维护成本：统一的审计字段便于数据追踪和问题排查
